Internal UI Framework

Improving the current and future UI

Testing, research, interaction design

The project

An internal UI development framework meant to (1) help UX designers and business analysts design interfaces that have a uniform look-and-feel across products and (2) dramatically reduce development time.

The problem

The development team had built several releases of the framework without input from the UX team. As a result, the interface was not very user-friendly, and did not use the corporate design pattern library. In addition, the development team needed business and user requirements for features they would be building in the next release (v1.6), as well as dashboard designs for the release they were working on (v1.5).

The solution

To address the usability issues, two other UX designers and I conducted an expert review of the existing UI (v1.4), and I ran usability tests with several business analysts. I also co-led a small internal workstream that documented the framework patterns for the UX team and analyzed the gaps between the corporate pattern library and the framework patterns. As part of gathering requirements, I conducted a literature review of the company's product documentation to determine the alert functionality needed for v1.6.

For the dashboard design, I took a preliminary prototype from the UX lead and updated it to meet the release's requirements. One requirement was that the user be able to have multiple dashboards; my proposal to implement this functionality using tabs was adopted, and I added them to the prototype we delivered to Development.
